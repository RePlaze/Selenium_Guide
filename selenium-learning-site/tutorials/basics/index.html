1. Introduction to Selenium WebDriver

Selenium WebDriver is a powerful tool for automating web browsers. It provides a programming interface to create and execute test scripts for web applications.

What You'll Learn

  • Set up Selenium WebDriver with Java
  • Write your first automation script
  • Locate web elements using various strategies
  • Perform common browser actions
  • Handle synchronization issues with waits

Prerequisites

  • Basic knowledge of Java programming
  • JDK 8 or higher installed
  • An IDE (IntelliJ IDEA, Eclipse, or VS Code)
  • Chrome or Firefox browser

2. Environment Setup

Step 1: Create a Maven Project

First, create a new Maven project in your IDE. This will help manage dependencies easily.

pom.xml 
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

    <modelVersion>4.0.0</modelVersion>
    
    <groupId>com.seleniumhub</groupId>
    <artifactId>selenium-basics</artifactId>
    <version>1.0.0</version>
    
    <properties>
        <maven.compiler.source>11</maven.compiler.source>
        <maven.compiler.target>11</maven.compiler.target>
    </properties>
    
    <dependencies>
        <!-- Selenium WebDriver -->
        <dependency>
            <groupId>org.seleniumhq.selenium</groupId>
            <artifactId>selenium-java</artifactId>

        <dependency>
            <groupId>io.github.bonigarcia</groupId>
            <artifactId>webdrivermanager</artifactId>
            <version>5.6.2</version>
        </dependency>
        
        <!-- TestNG -->
        <dependency>
            <groupId>org.testng</groupId>
            <artifactId>testng</artifactId>
            <version>7.8.0</version>
            <scope>test</scope>
        </dependency>
    </dependencies>
</project>

Step 2: Understanding WebDriverManager

WebDriverManager automatically downloads and manages driver executables (chromedriver, geckodriver, etc.) for you.

Pro Tip: WebDriverManager eliminates the need to manually download and set up driver executables, saving time and preventing version mismatch issues.

3. Your First Selenium Script

Let's write a simple script that opens Google and performs a search.

FirstSeleniumTest.java 
import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;
import io.github.bonigarcia.wdm.WebDriverManager;

public class FirstSeleniumTest {
    public static void main(String[] args) {
        // Setup ChromeDriver using WebDriverManager
        WebDriverManager.chromedriver().setup();
        
        // Create a new instance of ChromeDriver
        WebDriver driver = new ChromeDriver();
        
        try {
            // Navigate to Google
            driver.get("https://www.google.com");
            
            // Find the search box
            WebElement searchBox = driver.findElement(By.name("q"));
            
            // Type in the search box
            searchBox.sendKeys("Selenium WebDriver");
            
            // Submit the search
            searchBox.submit();
            
            // Wait a bit to see the results
            Thread.sleep(3000);
            
            // Print the page title
            System.out.println("Page title is: " + driver.getTitle());
            
        } catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            // Close the browser
            driver.quit();
        }
    }
}

4. Finding Elements - Locator Strategies

Selenium provides multiple ways to locate elements on a web page. Choosing the right locator strategy is crucial for stable tests.

Common Locator Types

By ID

Most reliable and fastest

driver.findElement(By.id("submit-button"))

By Name

Good for form elements

driver.findElement(By.name("username"))

By Class Name

For elements with specific classes

driver.findElement(By.className("btn-primary"))

By CSS Selector

Powerful and flexible

driver.findElement(By.cssSelector("div.container > input[type='text']"))

By XPath

Can locate any element

driver.findElement(By.xpath("//button[text()='Submit']"))

By Link Text

For anchor tags

driver.findElement(By.linkText("Click Here"))

Locator Best Practices

  • Priority Order: ID > Name > CSS > XPath
  • Avoid: Absolute XPath, dynamic IDs, index-based locators
  • Use: Unique attributes, data-testid attributes, relative XPath
  • Test: Verify locators work across different browsers
LocatorExamples.java 
import org.openqa.selenium.*;
import org.openqa.selenium.chrome.ChromeDriver;
import io.github.bonigarcia.wdm.WebDriverManager;
import java.util.List;

public class LocatorExamples {
    public static void main(String[] args) {
        WebDriverManager.chromedriver().setup();
        WebDriver driver = new ChromeDriver();
        
        try {
            driver.get("https://example.com/form");
            
            // Finding single element
            WebElement emailField = driver.findElement(By.id("email"));
            emailField.sendKeys("test@example.com");
            
            // Finding multiple elements
            List buttons = driver.findElements(By.tagName("button"));
            System.out.println("Found " + buttons.size() + " buttons");
            
            // Using CSS Selector with pseudo-classes
            WebElement firstInput = driver.findElement(
                By.cssSelector("form input:first-child")
            );
            
            // Using XPath with conditions
            WebElement submitButton = driver.findElement(
                By.xpath("//button[@type='submit' and contains(text(), 'Submit')]")
            );
            
            // Chaining locators (finding element within element)
            WebElement form = driver.findElement(By.id("signup-form"));
            WebElement formSubmitBtn = form.findElement(By.cssSelector("button[type='submit']"));
            
        } finally {
            driver.quit();
        }
    }
}

5. Common Browser Actions

Selenium provides various methods to interact with web elements and control the browser.

Element Interactions

ElementActions.java 
import org.openqa.selenium.*;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.support.ui.Select;
import io.github.bonigarcia.wdm.WebDriverManager;

public class ElementActions {
    public static void main(String[] args) {
        WebDriverManager.chromedriver().setup();
        WebDriver driver = new ChromeDriver();
        
        try {
            driver.get("https://example.com/demo");
            
            // Click action
            WebElement button = driver.findElement(By.id("submit-btn"));
            button.click();
            
            // Type text
            WebElement textField = driver.findElement(By.name("username"));
            textField.clear(); // Clear existing text
            textField.sendKeys("john.doe");
            
            // Select from dropdown
            WebElement dropdown = driver.findElement(By.id("country"));
            Select select = new Select(dropdown);
            select.selectByVisibleText("United States");
            select.selectByValue("US");
            select.selectByIndex(0);
            
            // Checkbox and Radio buttons
            WebElement checkbox = driver.findElement(By.id("terms"));
            if (!checkbox.isSelected()) {
                checkbox.click();
            }
            
            // Get element properties
            String buttonText = button.getText();
            String inputValue = textField.getAttribute("value");
            boolean isDisplayed = button.isDisplayed();
            boolean isEnabled = button.isEnabled();
            
            // Submit form
            WebElement form = driver.findElement(By.tagName("form"));
            form.submit();
            
        } finally {
            driver.quit();
        }
    }
}

6. Waits & Synchronization

Proper synchronization is crucial for stable test automation. Selenium provides different types of waits to handle dynamic content.

Types of Waits

Implicit Wait

Global timeout for all elements

driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));

✓ Simple to implement

✗ Applies to all elements

Explicit Wait

Wait for specific conditions

WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("result")));

✓ Flexible and precise

✓ Multiple conditions

Fluent Wait

Customizable polling and exceptions

Wait wait = new FluentWait<>(driver)
    .withTimeout(Duration.ofSeconds(30))
    .pollingEvery(Duration.ofSeconds(5))
    .ignoring(NoSuchElementException.class);

✓ Most flexible

✗ More complex

Common Wait Conditions

WaitExamples.java 
import org.openqa.selenium.support.ui.WebDriverWait;
import org.openqa.selenium.support.ui.ExpectedConditions;
import java.time.Duration;

public class WaitExamples {
    public static void waitExamples(WebDriver driver) {
        WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
        
        // Wait for element to be visible
        WebElement element = wait.until(
            ExpectedConditions.visibilityOfElementLocated(By.id("loading-complete"))
        );
        
        // Wait for element to be clickable
        WebElement button = wait.until(
            ExpectedConditions.elementToBeClickable(By.id("submit-button"))
        );
        
        // Wait for text to be present
        wait.until(
            ExpectedConditions.textToBePresentInElementLocated(
                By.id("status"), "Success"
            )
        );
        
        // Wait for element to disappear
        wait.until(
            ExpectedConditions.invisibilityOfElementLocated(By.id("spinner"))
        );
        
        // Wait for alert
        Alert alert = wait.until(ExpectedConditions.alertIsPresent());
        
        // Custom condition
        wait.until(driver -> {
            WebElement elem = driver.findElement(By.id("dynamic-content"));
            return elem.getText().length() > 0;
        });
        
        // Wait for page load (JavaScript)
        wait.until(webDriver -> 
            ((JavascriptExecutor) webDriver)
                .executeScript("return document.readyState")
                .equals("complete")
        );
    }
}

7. Best Practices

Use Page Object Model

Separate test logic from page structure for maintainability

Smart Waits

Use explicit waits instead of Thread.sleep()

Reliable Locators

Use stable attributes and avoid brittle XPath

Clean Up

Always quit the driver in finally block or @AfterMethod

Exception Handling

Handle common exceptions gracefully

Logging

Add meaningful logs for debugging

Example: Page Object Model

LoginPage.java 
public class LoginPage {
    private WebDriver driver;
    private WebDriverWait wait;
    
    // Locators
    private By usernameField = By.id("username");
    private By passwordField = By.id("password");
    private By loginButton = By.id("login-btn");
    private By errorMessage = By.className("error-msg");
    
    public LoginPage(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
    }
    
    public void enterUsername(String username) {
        wait.until(ExpectedConditions.visibilityOfElementLocated(usernameField))
            .sendKeys(username);
    }
    
    public void enterPassword(String password) {
        driver.findElement(passwordField).sendKeys(password);
    }
    
    public void clickLogin() {
        wait.until(ExpectedConditions.elementToBeClickable(loginButton))
            .click();
    }
    
    public boolean isErrorDisplayed() {
        try {
            return driver.findElement(errorMessage).isDisplayed();
        } catch (NoSuchElementException e) {
            return false;
        }
    }
    
    public HomePage login(String username, String password) {
        enterUsername(username);
        enterPassword(password);
        clickLogin();
        return new HomePage(driver);
    }
}
